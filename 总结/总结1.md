## 一、项目概述

### 1.1 项目定位

AI SSH/FTP Proxy 是一个专为 AI Agent 设计的远程服务器操作代理服务。它在目标服务器上运行，
将 SSH 命令执行、文件传输等操作封装为 HTTP API 和 WebSocket 接口，使 AI Agent 无需直接
管理 SSH 连接即可安全高效地操控远程服务器。

### 1.2 核心能力

| 能力 | 接口 | 说明 |
|------|------|------|
| SSH 命令执行 | `POST /api/ssh/exec` | Base64 编码的命令，同步返回结果 |
| SSH GET 执行 | `GET /api/ssh/exec?cmd=` | 避开 PowerShell JSON 转义问题 |
| SSH 异步执行 | `POST /api/ssh/exec/async` | 长任务后台执行，轮询获取结果 |
| SSH 脚本执行 | `POST /api/ssh/script` | 执行完整 bash 脚本或批量命令 |
| 文件上传 | `POST /api/file/upload` | HTTP Multipart，支持自动解压 |
| 文件列表 | `POST /api/file/list` | 列出目录内容 |
| 文件下载 | `POST /api/file/download` | 下载单个文件 |
| 文件删除 | `POST /api/file/delete` | 删除文件/目录 |
| 批量删除 | `POST /api/file/batch/delete` | 批量删除多个文件 |
| 创建目录 | `POST /api/file/mkdir` | 递归创建目录 |
| 重命名/移动 | `POST /api/file/rename` | 文件重命名或移动 |
| 复制 | `POST /api/file/copy` | 文件/目录复制 |
| 文件信息 | `POST /api/file/info` | 获取文件详细信息 |
| FTP 列表 | `POST /api/ftp/list` | FTP 目录列表 |
| FTP 上传 | `POST /api/ftp/upload` | FTP 文件上传 |
| FTP 下载 | `POST /api/ftp/download` | FTP 文件下载 |
| 健康检查 | `GET /api/health` | 服务状态 + 版本号 |
| 交互式 Shell | `ws://SERVER:48892/ws/ssh` | WebSocket 实时终端 |

### 1.3 技术栈

| 技术 | 用途 | 版本/说明 |

## 四、架构设计

### 4.1 分层架构

```
┌─────────────────────────────────────────────────────┐
│                    AI Agent (调用方)                   │
│              PowerShell / curl / SDK                  │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP / WebSocket
┌──────────────────────▼──────────────────────────────┐
│                  Server Layer (Gin)                    │
│  ┌──────────┐ ┌──────────────┐ ┌──────────────────┐ │
│  │ http.go  │ │ftp_handler.go│ │  websocket.go    │ │
│  │ SSH API  │ │  FTP API     │ │  WS SSH Shell    │ │
│  │ File API │ │              │ │                  │ │
│  │ Async    │ │              │ │                  │ │
│  │ Script   │ │              │ │                  │ │
│  └────┬─────┘ └──────┬───────┘ └────────┬─────────┘ │
│       │              │                   │           │
│  ┌────▼─────┐  ┌─────▼──────┐  ┌────────▼─────────┐ │
│  │ SSH Svc  │  │  FTP Svc   │  │   File Svc       │ │
│  │ Exec()   │  │  List()    │  │   Upload()       │ │
│  │ Interact │  │  Upload()  │  │   Extract()      │ │
│  │          │  │  Download()│  │   Mkdir/Copy/... │ │
│  └────┬─────┘  └─────┬──────┘  └────────┬─────────┘ │
└───────┼──────────────┼──────────────────┼───────────┘
        │              │                  │
┌───────▼──────────────▼──────────────────▼───────────┐
│              Target Server (Linux)                    │
│         SSH (port 22)    FTP (port 21)               │
│         File System      Shell                       │
└─────────────────────────────────────────────────────┘
```

### 4.2 配置优先级

```
环境变量 (SFTP_*) > config.yaml > 内置默认值
```

支持两套环境变量前缀：
- `SFTP_SSH_HOST`, `SFTP_SSH_PORT`, `SFTP_SSH_PASS` ... (简洁版)
- `APP_SSH_SERVER_HOST`, `APP_SSH_SERVER_PORT` ... (Viper 自动映射)

### 4.3 数据编码约定

所有 API 的输入输出均使用 **Base64 编码**：
- 命令 → Base64 编码后传入
- stdout/stderr → Base64 编码后返回
- 文件路径 → Base64 编码后传入
- 原因：防止 JSON 注入、特殊字符转义问题、二进制安全

### 4.4 中间件链

```
Request → CompatibilityMiddleware → LoggerMiddleware → Handler → Response
           (Connection:close)       (access log)
```

- `CompatibilityMiddleware`：设置 `Connection: close`，修复 PowerShell `Invoke-RestMethod` 挂起问题
- `LoggerMiddleware`：记录每个请求的方法、路径、状态码、耗时

## 五、编译与发布

### 5.1 编译命令

```powershell
# Linux amd64
$env:CGO_ENABLED='0'; $env:GOOS='linux'; $env:GOARCH='amd64'
go build -o ssh-ftp-proxy ./cmd/server

# Windows amd64
$env:CGO_ENABLED='0'; $env:GOOS='windows'; $env:GOARCH='amd64'
go build -o ssh-ftp-proxy.exe ./cmd/server
```

### 5.2 发布流程

```powershell
# 1. 提交代码
git add -A; git commit -m "feat: v1.x.0 - description"
git push origin main

# 2. 创建 tag
git tag -a v1.x.0 -m "v1.x.0 - description"
git push origin v1.x.0

# 3. 创建 GitHub Release（会上传 ~63MB 附件，需要等待）
gh release create v1.x.0 --title "v1.x.0 - 标题" --notes "changelog" --latest \
  ssh-ftp-proxy ssh-ftp-proxy.exe scripts/install.sh scripts/ssh_exec.ps1 \
  scripts/deploy.sh scripts/deploy.ps1 scripts/manage.sh
```

### 5.3 部署到服务器

```powershell
# 1. 停止服务（通过 SSH）
ssh -p 4306 root@23.159.8.47 "systemctl stop ssh-ftp-proxy"

# 2. 上传新二进制
scp -P 4306 ssh-ftp-proxy root@23.159.8.47:/opt/ssh-ftp-proxy/

# 3. 重启服务
ssh -p 4306 root@23.159.8.47 "systemctl start ssh-ftp-proxy"
```
