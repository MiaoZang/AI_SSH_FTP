## 一、核心代码详解

### 1.1 程序入口 - cmd/server/main.go

```go
// 文件路径: cmd/server/main.go
// 行数: 84 行
// 职责: 程序入口，初始化配置、日志、服务器，处理优雅关闭
```

**启动流程：**
1. 解析命令行参数（`-config` 配置文件路径）
2. 加载配置（支持文件 + 环境变量 + 默认值）
3. 初始化日志系统（slog + 文件输出）
4. 启动 WebSocket 服务器（异步，端口 48892）
5. 启动 HTTP 服务器（异步，端口 48891）
6. 监听 SIGINT/SIGTERM 信号，优雅关闭

### 1.2 配置系统 - internal/config/config.go

```go
// 文件路径: internal/config/config.go
// 行数: ~130 行
// 职责: 三层配置加载（默认值 → 环境变量 → 配置文件）
```

**核心结构体：**

```go
type Config struct {
    Server    ServerConfig  // HTTP/WS 端口、绑定 IP
    SSHServer SSHConfig     // SSH 连接信息
    FTPServer FTPConfig     // FTP 连接信息
    Log       LogConfig     // 日志级别、文件路径
}
```

**三层配置优先级（高 → 低）：**

| 层级 | 来源 | 示例 |
|------|------|------|
| 1. 环境变量 (SFTP_*) | `SFTP_SSH_HOST=127.0.0.1` | 最适合 AI 一行启动 |
| 2. 配置文件 | `config/config.yaml` | 传统部署方式 |
| 3. 内置默认值 | 代码硬编码 | HTTP=48891, SSH=22 |

**环境变量映射表：**

| 环境变量 | 配置键 | 默认值 |
|----------|--------|--------|
| `SFTP_SSH_HOST` | ssh_server.host | 127.0.0.1 |
| `SFTP_SSH_PORT` | ssh_server.port | 22 |
| `SFTP_SSH_USER` | ssh_server.user | root |
| `SFTP_SSH_PASS` | ssh_server.password | (空) |
| `SFTP_SSH_KEY` | ssh_server.key_file | (空) |
| `SFTP_HTTP_PORT` | server.http_port | 48891 |
| `SFTP_WS_PORT` | server.ws_port | 48892 |
| `SFTP_BIND_IP` | server.bind_ip | 0.0.0.0 |
| `SFTP_FTP_HOST` | ftp_server.host | 同 SSH |
| `SFTP_FTP_PORT` | ftp_server.port | 21 |
| `SFTP_FTP_USER` | ftp_server.user | 同 SSH |
| `SFTP_FTP_PASS` | ftp_server.password | 同 SSH |
| `SFTP_LOG_LEVEL` | log.level | info |
| `SFTP_LOG_FILE` | log.file | logs/server.log |

**智能默认：** FTP 配置未显式设置时，自动继承 SSH 的 host/user/password

### 1.3 HTTP路由 - internal/server/http.go

```go
// 文件路径: internal/server/http.go
// 行数: 700+ 行（最大文件）
// 职责: 所有 HTTP API 路由注册 + 请求处理器
```

**路由注册（setupRoutes）：**

```
GET  /api/health              → handleHealth
POST /api/ssh/exec            → handleSSHExec (同步执行)
GET  /api/ssh/exec?cmd=       → handleSSHExecGet (GET 方式执行)
POST /api/ssh/exec/async      → handleSSHExecAsync (异步执行)
GET  /api/ssh/task/:id        → handleSSHTaskStatus (查询异步任务)
POST /api/ssh/script          → handleSSHScript (脚本/批量执行)
POST /api/ftp/list            → handleFTPList
POST /api/ftp/upload          → handleFTPUpload
POST /api/ftp/download        → handleFTPDownload
POST /api/file/upload         → handleFileUpload (支持解压)
POST /api/file/list           → handleFileList
POST /api/file/download       → handleFileDownload
POST /api/file/delete         → handleFileDelete
POST /api/file/mkdir          → handleFileMkdir
POST /api/file/rename         → handleFileRename
POST /api/file/copy           → handleFileCopy
POST /api/file/info           → handleFileInfo
POST /api/file/batch/delete   → handleFileBatchDelete
```

**中间件：**

| 中间件 | 功能 | 解决的问题 |
|--------|------|------------|
| `CompatibilityMiddleware` | 设置 `Connection: close` 头 | PowerShell Invoke-RestMethod 挂起 |
| `LoggerMiddleware` | 记录请求日志（方法/路径/状态/耗时） | 可观测性 |

**异步任务系统：**

```go
type AsyncTask struct {
    ID        string           // task_12345_1
    Status    string           // running / done / error
    Command   string           // 原始命令
    Result    *SSHExecResponse // 执行结果
    CreatedAt time.Time        // 创建时间
    DoneAt    *time.Time       // 完成时间
}
```

任务存储在内存 `sync.Map` 中（重启后丢失），通过 `nextTaskID()` 生成唯一 ID

### 1.4 SSH 服务 - internal/service/ssh/service.go

```go
// 文件路径: internal/service/ssh/service.go
// 行数: ~80 行
// 职责: SSH 连接管理、命令执行
```

**核心方法：**

```go
// Exec 执行单条命令，返回 stdout/stderr/exitCode/error
func (s *Service) Exec(cmd string) (string, string, int, error)
```

- 每次 Exec 创建新的 SSH Session（避免连接竞争）
- 使用密码认证或密钥认证
- 返回值均为原始字符串（调用方负责 Base64 编码）

### 1.5 文件操作服务 - internal/service/file/service.go

```go
// 文件路径: internal/service/file/service.go
```

### 2.2 SSH 同步执行

```
POST /api/ssh/exec
Content-Type: application/json
```

**请求体：**
```json
{
  "command": "bHMgLWxhIC8="    // Base64("ls -la /")
}
```

**响应：**
```json

  "stdout": "dG90YWwgODQK...",  // Base64 编码的标准输出

}
```


**参数：** `cmd` = Base64 编码的命令

### 2.7 文件上传（支持自动解压）

```
POST /api/file/upload
Content-Type: multipart/form-data
```

**表单字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `file` | File | ✅ | 要上传的文件 |
| `path` | String | ✅ | Base64 编码的目标路径 |
| `extract` | String | ❌ | 设为 `true` 自动解压（tar.gz/zip/tar） |

**示例：**
```bash

  -F "file=@dist.tar.gz" \
  -F "path=$DEST" \
  -F "extract=true"
```

## 三、PowerShell 使用指南

### 3.1 推荐：Invoke-SSHCommand 函数

```powershell
function Invoke-SSHCommand {
    param(
        [Parameter(Mandatory)][string]$Command,
        [string]$Server = "http://SERVER:48891",
        [int]$Timeout = 300
    )
    $b64 = [Convert]::ToBase64String([Text.Encoding]::UTF8.GetBytes($Command))
    $resp = Invoke-RestMethod -Uri "$Server/api/ssh/exec?cmd=$b64" -Method GET -TimeoutSec $Timeout
    $out = if ($resp.stdout) { [Text.Encoding]::UTF8.GetString([Convert]::FromBase64String($resp.stdout)) } else { "" }
    $err = if ($resp.stderr) { [Text.Encoding]::UTF8.GetString([Convert]::FromBase64String($resp.stderr)) } else { "" }
    return @{ stdout = $out; stderr = $err; exit_code = $resp.exit_code }
}
```

**使用示例：**
```powershell
$r = Invoke-SSHCommand "ls -la /"
Write-Host $r.stdout

$r = Invoke-SSHCommand "cat /etc/os-release"
Write-Host $r.stdout
```

### 3.2 已知注意事项

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| JSON 转义失败 | PowerShell `$` 和 `"` 冲突 | 使用 GET API 或 ssh_exec.ps1 |
| Invoke-RestMethod 挂起 | Keep-Alive 不兼容 | CompatibilityMiddleware 已修复 |
| 变量残留乱序 | 同一会话变量作用域共享 | 每次用 `$r = Invoke-SSHCommand` 隔离 |
| 多行命令 heredoc 报错 | `@"..."@` 对 `$` 敏感 | 使用 `@'...'@` 或 /api/ssh/script |
