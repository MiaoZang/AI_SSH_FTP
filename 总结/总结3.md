## 一、版本历史

### v1.0.0 (2026-01-18) - 初始版本

**提交：** `2ea4e05 Initial commit: AI SSH/FTP Proxy Service v1.0.0`

**功能：**
- SSH 命令执行 API（POST /api/ssh/exec）
- FTP 文件操作（list/upload/download）
- WebSocket 交互式 SSH Shell
- 配置文件驱动（config.yaml）
- 基础日志系统

**架构：**
```
cmd/server/main.go → Server(Gin) → SSHService / FTPService
```

---

### v1.1.0 (2026-02-08) - 稳定性修复

**提交：** `372fa87 Fix: SSH race condition, WS goroutine leak, graceful shutdown`

**修复项：**
- SSH 连接竞争条件（多个请求同时使用同一 SSH Session）
- WebSocket goroutine 泄漏（断开连接后 goroutine 未回收）
- 优雅关闭机制（SIGINT/SIGTERM 信号处理）
- Base64 编码统一化

**教训：** Go 的 SSH Session 不是线程安全的，每次 Exec() 应创建新 Session

---

### v1.2.0 (2026-02-09) - HTTP 文件上传

**提交：** `86fcb7e feat: Add HTTP file upload API with multipart support`

**新功能：**
- `POST /api/file/upload` — HTTP Multipart 文件上传
- 自动解压支持（tar.gz, zip, tar）
- `extract=true` 参数控制是否解压
- 路径末尾 `/` 自动附加文件名

**设计决策：**
- 选择 HTTP Multipart 而非 FTP，因为 AI Agent 调用 HTTP 更简单
- 解压通过 SSH 执行服务器端 `tar xzf` / `unzip` 命令实现
- 上传文件先通过 SFTP 写入临时位置，再移动到目标路径

**后续修复：**
- `b96c3c7` - 修复目录路径末尾 `/` 处理
- `f726da7` - 修复 deploy.ps1 编码问题（改用 ASCII）

---

### v1.3.0 (2026-02-10) - 文件操作扩展

**提交：** `96fc90f feat(P0): add file operations APIs`

**新功能：**

| API | 底层实现 | 说明 |
|-----|----------|------|
| `POST /api/file/mkdir` | `mkdir -p` | 递归创建目录 |
| `POST /api/file/rename` | `mv` | 重命名/移动 |
| `POST /api/file/copy` | `cp -r` | 复制文件/目录 |
| `POST /api/file/info` | `stat` | 获取文件详情 |
| `POST /api/file/batch/delete` | 循环 `rm -rf` | 批量删除 |

**设计决策：**
- 所有文件操作通过 SSH 命令实现（而非 SFTP 协议），保持架构统一
- 路径参数统一使用 Base64 编码

## 二、三种部署模式详解

### 模式 A：install.sh 自动安装（推荐 AI）

```bash
# SSH 到服务器后执行
curl -fsSL https://raw.githubusercontent.com/MiaoZang/AI_SSH_FTP/main/scripts/install.sh \
  -o /tmp/install.sh && chmod +x /tmp/install.sh && \
  /tmp/install.sh --ssh-port 4306 --ssh-user root --ssh-pass PASSWORD --auto-start --systemd
```

**优点：** 全自动、支持 systemd、有健康检查
**缺点：** 需要 GitHub 网络连通

### 模式 B：手动安装（fallback）

```bash
mkdir -p /opt/ssh-ftp-proxy/{config,logs,scripts}
cd /opt/ssh-ftp-proxy
wget -q https://github.com/MiaoZang/AI_SSH_FTP/releases/latest/download/ssh-ftp-proxy
chmod +x ssh-ftp-proxy
cat > config/config.yaml << EOF
server:
  http_port: 48891
  ws_port: 48892
  bind_ip: "0.0.0.0"
ssh_server:
  host: "127.0.0.1"
  port: 4306
  user: "root"
  password: "PASSWORD"
EOF
wget -q https://raw.githubusercontent.com/MiaoZang/AI_SSH_FTP/main/scripts/manage.sh \
  -O scripts/manage.sh && chmod +x scripts/manage.sh
bash scripts/manage.sh start
```

**优点：** 完全可控、不依赖 install.sh
**缺点：** 步骤多、需手动创建配置

### 模式 C：环境变量启动（最简）

```bash
mkdir -p /opt/ssh-ftp-proxy/logs && cd /opt/ssh-ftp-proxy
wget -q https://github.com/MiaoZang/AI_SSH_FTP/releases/latest/download/ssh-ftp-proxy
chmod +x ssh-ftp-proxy
SFTP_SSH_PORT=4306 SFTP_SSH_USER=root SFTP_SSH_PASS=PASSWORD \
  nohup ./ssh-ftp-proxy > logs/server.log 2>&1 &
```

**优点：** 最少文件（只需二进制）、最少步骤
**缺点：** 无 systemd 自动重启

### 模式对比

| 特性 | 模式 A | 模式 B | 模式 C |
|------|--------|--------|--------|
| 需要文件数 | 1 (install.sh) | 3 (binary+config+manage) | 1 (binary) |
| 需要手动步骤 | 1 步 | 5+ 步 | 3 步 |
| systemd 支持 | ✅ | 需手动 | ❌ |
| 配置文件 | 自动生成 | 手动创建 | 不需要 |
| GitHub 依赖 | 需要 | 需要 | 需要 |
| AI 友好度 | ★★★★★ | ★★★ | ★★★★ |
