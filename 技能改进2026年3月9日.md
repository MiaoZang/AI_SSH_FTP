# ai-ssh-ftp-proxy 技能改进建议

> 更新时间：2026-03-09 06:05

## 一、本次部署遇到的难点

### 1. 交互式脚本无法被 AI 正确操作
- **问题**：`manage.sh` 的 `curl ... | bash` 安装方式会进入交互式菜单（选项 1-8），AI 子代理无法正确识别菜单项，反复选择"选项6 重新配置"，陷入死循环。
- **根因**：宝塔终端是一个 Web Terminal（xterm），浏览器子代理通过逐键输入操作，无法可靠地读取终端输出和判断菜单上下文。
- **解决**：阅读 `manage.sh` 源码后发现支持 CLI 参数（`manage.sh start|stop|restart|status`），手动创建 `config.yaml` 跳过交互式配置向导，再用 `bash manage.sh start` 免交互启动。

### 2. 技能文档缺少"AI 友好"的安装说明
- **问题**：SKILL.md 只写了 `curl ... | bash` 一行安装命令，没有说明免交互方式。
- **影响**：AI 无法自动判断应该先创建 `config.yaml` 再执行 `manage.sh start`。

### 3. 服务已存在导致端口冲突
- **问题**：服务器上已有旧进程（PID 78579，2月12日启动）占用 48891 端口，新启动的进程因 `bind: address already in use` 失败。
- **影响**：AI 子代理未能正确判断"服务已在运行"，仍然尝试重新安装。

### 4. Windows 环境无 sshpass
- **问题**：Windows PowerShell 没有 `sshpass`，无法通过管道传递密码给 SSH 命令。
- **影响**：直接 SSH 连接方式在 Windows 上受阻，最终改用宝塔面板终端和技能 API 两种替代方式。

## 二、技能改进建议

### A. SKILL.md 文档改进

#### 建议 1：增加"AI 免交互部署"章节
```markdown
### AI Non-Interactive Deployment (推荐 AI 使用)

# 1. 创建目录
mkdir -p /opt/ai-ssh-ftp-proxy/scripts /opt/ai-ssh-ftp-proxy/config

# 2. 下载管理脚本
cd /opt/ai-ssh-ftp-proxy
wget -q https://raw.githubusercontent.com/MiaoZang/AI_SSH_FTP/main/scripts/manage.sh -O scripts/manage.sh
chmod +x scripts/manage.sh

# 3. 手动创建配置（跳过交互向导）
cat > config/config.yaml << 'EOF'
server:
  http_port: 48891
  ws_port: 48892
  bind_ip: "0.0.0.0"
ssh_server:
  host: "127.0.0.1"
  port: 22
  user: "root"
  password: "YOUR_PASSWORD"
EOF

# 4. 免交互启动
bash scripts/manage.sh start
```

#### 建议 2：增加 CLI 参数说明
```markdown
### manage.sh CLI 参数（免交互模式）
| 命令 | 功能 |
|------|------|
| `manage.sh start` | 启动服务 |
| `manage.sh stop` | 停止服务 |
| `manage.sh restart` | 重启服务 |
| `manage.sh status` | 查看运行状态 |
| `manage.sh logs` | 查看日志 |
| `manage.sh update` | 更新二进制 |
```

#### 建议 3：增加"连通性验证"示例
```markdown
### 验证部署（从本地 PowerShell）
$cmd = [Convert]::ToBase64String([Text.Encoding]::UTF8.GetBytes("whoami"))
$body = @{command = $cmd} | ConvertTo-Json
Invoke-RestMethod -Uri "http://SERVER:48891/api/ssh/exec" -Method POST -ContentType "application/json" -Body $body
```

### B. manage.sh 脚本改进

#### 建议 4：增加首次安装时的自动检测逻辑
```bash
# 在 main() 函数中增加：
if [[ ! -f "$CONFIG_FILE" ]] && [[ $# -eq 0 ]]; then
    echo "首次安装检测到，进入配置向导..."
    create_config
    start_service
    exit 0
fi
```

#### 建议 5：增加防重复启动检测
```bash
# 在 start_service() 中增加端口占用检测：
if netstat -tunlp 2>/dev/null | grep -q ":$http_port "; then
    echo "端口 $http_port 已被占用，可能服务已在运行"
    echo "使用 'manage.sh status' 查看状态"
    return 1
fi
```

### C. 技能架构改进

#### 建议 6：支持环境变量配置（最适合 AI）
```bash
# 允许通过环境变量传入配置，完全跳过文件和交互
SSH_HOST=127.0.0.1 SSH_PORT=22 SSH_USER=root SSH_PASS=xxx ./ssh-ftp-proxy
```

#### 建议 7：增加 /api/health 健康检查端点
```bash
curl http://SERVER:48891/api/health
# 返回 {"status": "ok", "uptime": "2h30m", "version": "1.2.0"}
```
